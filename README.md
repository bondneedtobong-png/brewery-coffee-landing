# Brewery Coffee — full-stack демо лендинга кафе

Демонстрационный портфолио-проект: тёмный амбер-лендинг крафтового кофе с **настоящим бэкендом**, реляционной БД, регистрацией по телефону, программой лояльности на 10 уровней, отзывами с модерацией и админкой с разграничением ролей.

> Витрина превратилась в живую демку: любой заказчик с первого клика видит, что у сайта есть БД, а не только красивые анимации.

## Демо

- **Live:** *(после деплоя на Render — см. ниже)*
- **GitHub:** https://github.com/bondneedtobong-png/brewery-coffee-landing

## Стек

- **Node.js 22+** (используется встроенный `node:sqlite`)
- **Express 4** — роутинг, сессии (`express-session`)
- **EJS** — серверный рендеринг страниц
- **SQLite** — реляционная БД, файл `data/app.db`
- **bcryptjs** — хеш паролей
- **Чистый CSS / Vanilla JS** для фронта — никаких bundler'ов

## Запуск локально

```bash
git clone https://github.com/bondneedtobong-png/brewery-coffee-landing.git
cd brewery-coffee-landing
npm install
npm run setup     # миграции + сид-данные
npm start         # http://localhost:3000
```

Сервер поднимется на `http://localhost:3000`. SQLite-файл лежит в `data/app.db`. Чтобы пересоздать с нуля:

```bash
npm run reset
```

## Что есть на сайте

### Публичный лендинг (`/`)
- Hero со своей фотографией кофейного пакета, бейджем «№1»
- Секция «О нас» с большим фото зала, четырьмя плитками-фичами, статистикой
- **Барная карта из БД** — типографическая, без картинок, только напитки. Эспрессо + альтернатива, бейджи «Хит»/«Новинка», теги
- **Отзывы из БД** — карточки с агрегатным рейтингом и количеством, фоновые фото с blur-эффектом
- Бронирование стола с **интерактивной SVG-схемой круга мест**, форма автоматически предзаполняется именем и телефоном залогиненного пользователя
- Контакты с пульсирующим пином и стилизованной картой
- CTA «вступить в клуб» — меняется на «открыть кабинет» когда залогинен

### Регистрация и личный кабинет
- Регистрация по **номеру телефона** (нормализуется в E.164: `+7XXXXXXXXXX`), имени, e-mail (необязательно), паролю
- **Обязательный чекбокс согласия** на обработку ПД — сохраняется флаг + timestamp
- Пароль хешируется bcrypt (factor 10)
- В личном кабинете видно текущий уровень, % скидки, прогресс до следующего уровня, перки по диапазону, история начислений и собственные отзывы со статусом модерации
- Редактирование имени / телефона / e-mail

### Бонусная программа (10 уровней)
| Уровень | Сумма (₽) | Скидка | Перки |
|---------|-----------|--------|-------|
| 1 | 0 | 3% | 1-3: бесплатный кофе раз в неделю за тёплый отзыв |
| 2 | 3 000 | 5% | 1-3 |
| 3 | 7 000 | 7% | 1-3 |
| 4 | 12 000 | 9% | 4-9: бесплатный десерт + кофе раз в неделю |
| 5 | 20 000 | 11% | 4-9 |
| 6 | 30 000 | 13% | 4-9 |
| 7 | 45 000 | 15% | 4-9 |
| 8 | 65 000 | 17% | 4-9 |
| 9 | 90 000 | 19% | 4-9 |
| 10 | 120 000 | 20% | 10: ежедневный бесплатный кофе + еженедельный десерт |

Уровень привязан к **телефону** и **никогда не сбрасывается**. Прогресс считается из суммы транзакций.

### Отзывы
- Зарегистрированный гость оставляет отзыв (текст 20–1000 символов + оценка 1–5 звёзд)
- Статус `pending` по умолчанию — на сайте видны только `approved`
- Агрегаты: общее количество одобренных + средний рейтинг
- Звёзды кликабельные в форме (CSS-only radio rating)

### Админка (`/admin`)
RBAC: `user` / `moderator` / `admin`. Доступ только модераторам/админам.

- **Сводка** — счётчики по пользователям, меню, отзывам (с подсветкой ждущих модерации), событиям
- **Модерация отзывов** — три таба: на модерации / опубликованы / отклонены. Кнопки одобрить / отклонить. Удаление — только для админа
- **Меню** — добавить / изменить / удалить позицию, флаги Хит/Новинка, порядок
- **События** — создание/редактирование/удаление, статусы upcoming/past/cancelled
- **Пользователи** — список с уровнем и накопленной суммой. Админ может менять роль и начислить демо-сумму (автоматически пересчитывает уровень)

### Политика конфиденциальности (`/privacy`)
По духу 152-ФЗ: оператор, какие данные собираем, зачем, как храним (bcrypt, SQLite), сроки и удаление, права пользователя, передача третьим лицам (не передаём), cookies (только сессия).

## Деплой на Render

В проекте лежит готовый `render.yaml`. Достаточно:

1. Зайти на [render.com](https://render.com), залогиниться через GitHub
2. **New → Blueprint** → выбрать репозиторий `brewery-coffee-landing`
3. Render прочитает `render.yaml`, создаст Web Service + Persistent Disk (1GB, под SQLite)
4. Через 2–3 минуты получите URL вида `https://brewery-coffee-landing.onrender.com`

`render.yaml` уже задаёт:
- `NODE_VERSION=22`
- `NODE_ENV=production`
- `TRUST_PROXY=true` (Render — прокси)
- `SESSION_SECRET` — Render сгенерирует случайный
- `DB_PATH=/data/app.db` — постоянный диск
- buildCommand: `npm install && npm run setup` (миграции + сид при первом запуске)

> На бесплатном плане инстанс засыпает через ~15 минут бездействия и просыпается за 30 секунд при первом запросе. Постоянный диск сохраняет БД между перезапусками.
